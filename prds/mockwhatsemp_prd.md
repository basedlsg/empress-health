PRD — EmpressHealth.ai “Why Empress” Page Cleanup

Document Type: Frontend Cleanup PRD
Target Page (Mock): mockwhatempress
Folder: /mockpages
Path / Extension: /mockwhatsempress
Stack: Plain HTML / CSS / JavaScript (no framework)
Status: Design + layout cleanup only (no content changes)

1. Objective

The goal of this PRD is to visually refine and modernize the Why Empress page while preserving:

100% of existing copy (no text edits)

Original content order and hierarchy

Brand voice and intent

The cleanup should improve readability, flow, spacing, alignment, color harmony, and responsiveness across desktop and mobile.

2. Non-Negotiable Constraints

❌ Do not rewrite, summarize, reorder, or remove text

❌ Do not add new messaging or sections

❌ Do not change meaning, tone, or CTA language

❌ Do not introduce animations beyond subtle hover/focus states

✅ Layout, spacing, typography, and color refinements are allowed

✅ Semantic HTML improvements are allowed

3. Visual & Brand Guidelines (EmpressHealth)
3.1 Color System (Must Be Used)

Primary Purple: #3F1449

Accent Gold: #D8A738

Ivory / Soft Neutral: #D8D1CA

Text (Primary): Near-black or deep purple tint

Text (Secondary): Muted gray-violet tone

Usage rules:

Purple = authority & structure

Gold = emphasis, dividers, highlights

Ivory = breathing space, background sections

3.2 Typography Rules

Headlines: Elegant serif or refined display font

Body Text: Clean sans-serif for readability

Line Height:

Headlines: 1.2–1.3

Body: 1.6–1.8

Do not mix more than 2 font families total.

4. Layout Cleanup Requirements
4.1 Global Page Structure

Center content with a max-width container (1100–1200px desktop)

Consistent horizontal padding:

Desktop: 64px

Mobile: 20–24px

Clear vertical rhythm between sections (no stacking congestion)

4.2 Section Treatment

Each logical content block should:

Be visually separated using spacing, soft background shifts, or dividers

Avoid hard borders unless subtle (1px, low opacity)

Use consistent heading alignment (left-aligned preferred)

4.3 Headings & Subsections

Headings must clearly anchor each idea

Introduce visual hierarchy using:

Font size

Weight

Spacing (not color overload)

Gold accents may be used for:

Underlines

Small dividers

Icon bullets (if already present)

5. Readability & Flow Improvements

Limit text line width to ~65–75 characters

Break dense paragraphs using spacing (not new text)

Improve scannability by:

Increasing margin between paragraphs

Aligning text blocks consistently

Ensuring CTAs stand out visually

6. Call-to-Action (CTA) Treatment

CTAs must remain word-for-word unchanged

Improve visibility using:

Color contrast

Button sizing

Spacing from surrounding content

Primary CTA styling:

Purple background

Gold or ivory accent

Rounded corners (8–12px)

Clear hover/focus state

7. Responsive Behavior
7.1 Mobile (≤ 820px)

Stack all sections vertically

Reduce heading sizes proportionally

Maintain generous vertical spacing

Ensure buttons are full-width or easily tappable

7.2 Desktop

Avoid overly wide text blocks

Maintain a calm, editorial feel

No horizontal scrolling under any condition

8. Accessibility & UX

Color contrast must meet WCAG AA minimums

All buttons and links must have:

Hover state

Focus state

Semantic HTML:

Proper heading levels

Section landmarks where appropriate

9. Deliverable Requirements

File Name: mockwhatempress
Folder: /mockpages
Path: /mockwhatsempress

The output should be a single, self-contained mock page that visually represents the cleaned-up Why Empress page using the existing content.

10. Success Criteria

This PRD is considered successful if:

The page feels calmer, cleaner, and more premium

Content is easier to read and scan

Brand identity is reinforced visually

No text or meaning has been altered
