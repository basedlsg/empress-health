PRD — EmpressHealth.ai “Events” Page Cleanup

Document Type: Frontend Cleanup PRD
Source Page: https://empresshealth.ai/events

Target Mock Page: mockevents
Folder: /mockpages
Path / Extension: /mockevents
Stack: Plain HTML / CSS / JavaScript (no framework)
Status: Layout, visual hierarchy, and responsiveness cleanup only

1. Objective

Clean up and modernize the Events page to feel organized, premium, and intentional, while preserving:

All existing text content (100% unchanged)

Original content order and structure

Event information accuracy and hierarchy

The page should make it easy to:

Scan upcoming events

Understand event purpose, date, and participation context

Identify calls-to-action quickly

2. Non-Negotiable Constraints

❌ Do NOT edit, rewrite, summarize, reorder, or remove any text

❌ Do NOT add new events or new messaging

❌ Do NOT change CTA wording or intent

❌ Do NOT introduce complex animations

✅ You MAY improve layout, spacing, typography, color usage, alignment, and responsive behavior

✅ You MAY group existing event content visually (cards, sections) without changing order

3. EmpressHealth Brand System (Mandatory)
3.1 Color Palette

Primary Purple: #3F1449

Accent Gold: #D8A738

Ivory / Neutral Background: #D8D1CA

Primary Text: Near-black or deep purple tint

Secondary Text: Muted gray-violet

Usage rules:

Purple = structure & authority

Gold = emphasis (dates, dividers, CTA accents)

Ivory = background breathing room

3.2 Typography

Headings: Elegant serif or refined display font

Body: Clean sans-serif

Max 2 font families total

Line height:

Headings: 1.2–1.3

Body: 1.6–1.8

4. Page Structure & Layout Cleanup
4.1 Global Layout

Centered content container

Desktop max-width: 1100–1200px

Mobile: full width with padding

Horizontal padding:

Desktop: 64px

Mobile: 20–24px

Consistent vertical spacing between sections

4.2 Events Presentation

Each event should be visually grouped as a card or block

Cards must:

Preserve existing event order

Clearly surface:

Event title

Date / time (if present)

Description

CTA (if present)

Use subtle separation:

Soft background tint

Rounded corners (8–12px)

Thin dividers (optional)

No new data may be added to events.

4.3 Hierarchy & Scannability

Event titles must stand out clearly

Dates/times may use gold accents for visibility

Descriptions should be readable with controlled line width

CTAs should be visually distinct but restrained

5. CTA Styling Rules

CTA text must remain unchanged

Improve visibility using:

Purple background or outline

Gold accent (border, hover, or underline)

Rounded corners: 8–12px

Clear hover and focus-visible states

6. Readability Improvements

Limit text width for long descriptions (65–75 characters per line)

Increase spacing between paragraphs inside event cards

Avoid dense text blocks

Maintain a calm, editorial rhythm

7. Responsive Behavior
7.1 Mobile (≤ 820px)

Stack events vertically

Full-width event cards

Buttons large enough for touch

Maintain generous vertical spacing

7.2 Desktop

Single-column or clean two-column layout (if existing structure supports it)

No horizontal scrolling

Consistent alignment across event cards

8. Accessibility & UX

Color contrast must meet WCAG AA where possible

All interactive elements must include:

Hover state

Focus-visible outline

Use semantic HTML:

<main>, <section>, <article>

Proper heading levels per event

9. Deliverables

File Name: mockevents
Folder: /mockpages
Path: /mockevents

Single self-contained mock page

Inline CSS allowed

Minimal JS only if necessary

10. Success Criteria

The cleanup is successful if:

Events are easy to scan and understand

Page feels organized, premium, and on-brand

Mobile and desktop experiences feel intentional

No content or meaning has changed
