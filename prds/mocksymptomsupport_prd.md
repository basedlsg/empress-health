Product Requirements Document (PRD)
Project: EmpressHealth – Symptom Support Page Redesign

Page URL: https://empresshealth.ai/symptomsupport

New Mock Filename: mocksymptomsupport
Folder Location: /mockpages/mocksymptomsupport
Scope: UI/UX redesign only (no content edits)

1. Objective

Redesign the Symptom Support page to:

Align fully with the EmpressHealth brand system

Improve visual hierarchy and readability

Enhance mobile responsiveness

Elevate trust, calmness, and premium feel

Improve spacing, flow, and section clarity

Non-Goal:
❌ Do NOT change any wording
❌ Do NOT reorder sections
❌ Do NOT remove content
❌ Do NOT summarize or expand text

2. Design Principles

The page must feel:

Grounded

Elegant

Safe

Clean

Softly premium

Emotionally supportive

This page should visually communicate:

“You are seen. You are supported. You are guided.”

3. Brand System (Must Be Followed)
Primary Palette

Deep Plum: #3F1449

Royal Purple: #5A1E66

Gold Accent: #D8A738

Ivory Background: #F5EFE8

Soft Neutral: #D8D1CA

White: #FFFFFF

Typography

Headings:

Elegant serif (Playfair Display or similar)

Semi-bold or bold

Generous letter spacing

Body:

Clean sans-serif (Inter, Poppins, or similar)

16–18px desktop

15–16px mobile

Line-height 1.6–1.8

4. Layout Requirements
4.1 Overall Page Structure

Full-width hero section

Contained content width: max 1100px

Centered layout

Generous vertical spacing (80px desktop, 56px mobile)

Section padding: 40–80px

5. Section-by-Section Redesign
5.1 Hero Section
Current Issue:

Flat presentation

Insufficient emotional framing

Redesign Requirements:

Full-width ivory background

Subtle radial gradient overlay (plum fade)

Center-aligned content

Large elegant heading

Supporting subheading styled with softer tone

Optional subtle gold underline accent under main heading

CTA Styling:

Primary button:

Plum background

White text

Gold hover glow

12px border radius

Soft shadow

5.2 Content Blocks (Symptom Areas / Text Sections)
Current Issues:

Dense formatting

Insufficient separation between blocks

Redesign:

Each content grouping should be placed inside:

White card container

24–32px padding

20px border radius

Subtle shadow (very soft)

Gold top accent bar (2–4px)

Spacing:

40–60px between cards

5.3 Lists or Bullet Content

Increase spacing between bullets

Custom bullet style:

Small gold circle

Or minimal plum icon

Line height 1.7+

5.4 Callout or Guidance Sections

If there are guidance/support messaging blocks:

Light plum background

White text

Gold border accent

Rounded 20px corners

32px padding

Soft glow shadow

These sections should feel calming and reassuring.

5.5 Buttons / Links

All interactive elements must:

Have hover animation

Slight lift effect (translateY -2px)

Soft glow

Clear contrast ratios

Rounded corners (12px minimum)

6. Mobile Optimization
Required Adjustments:

Reduce padding proportionally

Stack all multi-column layouts

Increase tap targets (minimum 44px height)

Maintain readable font scaling

Remove excessive horizontal padding

Ensure buttons are full-width on mobile

7. Visual Enhancements (Optional but Allowed)

Allowed enhancements:

Subtle section dividers

Soft fade transitions between sections

Background texture overlay (very subtle grain)

Smooth scroll animations

Fade-in on scroll

Not allowed:

Changing any text

Rewriting headings

Altering content structure

8. Accessibility Requirements

WCAG AA compliant contrast

Minimum 16px body text

Clear focus states

Accessible button labeling

Proper semantic HTML structure

H1 only once (if currently structured that way — do not change hierarchy)

9. Technical Requirements

Framework Assumption:

Next.js (App Router)

Tailwind CSS

shadcn/ui components

File Requirements:

Create file:

/mockpages/mocksymptomsupport


File must:

Be fully responsive

Be visually polished

Mirror original content exactly

Use Empress theme colors

Be production-quality styling

10. Acceptance Criteria

The redesign is approved when:

All original text is preserved exactly

Layout feels premium and calm

Mobile experience is clean and readable

Clear visual hierarchy exists

Page aligns with Empress brand identity

Buttons and cards feel modern and interactive

11. Success Metrics (Design Quality Indicators)

Improved readability

Increased visual spacing clarity

Higher emotional alignment with brand

Stronger premium aesthetic

Cleaner mobile presentation
